# Claude 명령어: Review

현재 브랜치의 변경사항(`git diff`)을 분석하여 코드 리뷰 리포트를 생성합니다.

## 프로세스

1. `git diff`로 현재 변경사항 확인 (스테이지된 변경 + 언스테이지된 변경 모두)
2. 변경된 파일을 읽고 전체 맥락 파악
3. 아래 체크리스트 항목별로 점검
4. 요약 리포트 출력

## 체크리스트

### 🔒 보안
- XSS, SQL 인젝션, 커맨드 인젝션 등 OWASP Top 10 취약점
- 하드코딩된 시크릿, API 키, 비밀번호
- 사용자 입력값 검증 누락
- 인증/인가 로직 결함

### ⚡ 성능
- 불필요한 리렌더링 유발 패턴
- N+1 쿼리 또는 비효율적 데이터 페칭
- 번들 크기에 영향을 주는 불필요한 import
- 이미지 최적화 누락 (next/image 미사용)

### 🏗️ Next.js 컨벤션
- 서버/클라이언트 컴포넌트 분리 위반 ("use client"가 불필요하거나 누락)
- 서버 컴포넌트에서 브라우저 API(훅, 이벤트 핸들러) 사용
- 클라이언트 컴포넌트에서 서버 전용 기능 사용
- metadata export 누락
- `asChild` prop 사용 (이 프로젝트의 @base-ui/react 기반 shadcn에서 미지원)

### 🎨 스타일링
- Tailwind CSS v4 컨벤션 위반
- globals.css CSS 변수 미사용 (하드코딩된 색상값)
- 다크모드 미대응

### 📐 코드 품질
- TypeScript 타입 안전성 (any 사용, 타입 단언 남용)
- 에러 핸들링 누락
- 데드 코드 또는 미사용 import
- 네이밍 컨벤션 위반

## 리포트 포맷

```
## 코드 리뷰 리포트

### 요약
변경 파일 N개 | 이슈 N개 발견

### 🚨 Critical (즉시 수정 필요)
- [파일:라인] 설명

### ⚠️ Warning (개선 권장)
- [파일:라인] 설명

### 💡 Suggestion (선택 사항)
- [파일:라인] 설명

### ✅ Good
- 잘 작성된 부분 언급
```

## 규칙

- 이슈가 없는 카테고리는 생략
- 각 이슈에 파일명과 라인 번호를 반드시 포함
- 구체적인 수정 방안을 함께 제시
- 잘 작성된 부분도 반드시 언급하여 균형 잡힌 리뷰 제공
